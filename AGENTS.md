## 1. Project Overview

ROS2 camera node development template using notebook-driven development.
**Stack**: Python + uv + nbdev + ROS2 ($ROS_DISTRO) + OpenCV
**Source of truth**: `nbs/` — notebooks are the canonical source, not the exported `.py` files.

---

## 2. Orchestration Limits

- **ROS2 workspaces**: Never read, explore, or load information from ROS2 workspaces or packages. All relevant project knowledge is documented in the Jupyter notebooks in `nbs/`.
- **ROS2 scripts**: Never run `scripts/*.sh`, `scripts/*.zsh`, or generate workspaces/packages unless explicitly requested by the user.

---

## 3. Bootstrap Guard

Before doing ANY work, verify the project is bootstrapped:

- `.venv/` directory exists
- `pyproject.toml` exists

**Smart detection logic:**

| Condition | Action |
|-----------|--------|
| `pyproject.toml` + `uv.lock` exist | Scaffolding is uploaded to repo → run `uv sync` only |
| `.venv` exists but no `uv.lock` | Partial setup → run `uv sync` |
| Neither exists | No scaffolding at all → run full bootstrap |

**For full bootstrap**, tell the user to run:

```bash
bash scripts/basic_install.sh && bash scripts/prepare_project.bash
```

⚠️ **NEVER run bootstrap scripts on behalf of the user without explicit confirmation.**

## 6. Notebook Conventions

### Two types of notebooks

**Code/Module notebooks** (have `#| default_exp`):

```
Cell 1 (markdown): # Title\n> Description
Cell 2 (code):     #| default_exp module_name
Cell 3 (code):     #| hide\nfrom nbdev.showdoc import *
...code, demos, tests...
Last cell (code):  #| hide\nimport nbdev; nbdev.nbdev_export()
```

**Documentation-only notebooks** (NO `#| default_exp`):
- No executable code cells (or all marked `#| eval: false`)
- Pure markdown + instructions
- Example: `00_project_setup.ipynb`

### Naming convention

`NN_name.ipynb` — two-digit prefix for ordering.

## 7. Dependency Management

- Use `uv add <package>` — **NEVER `pip install`**
- For packages that might conflict with ROS2 system packages:
  ```bash
  uv add <package> --constraint system_constraints.txt
  ```
- ⚠️ `matplotlib-inline==0.1.6` is **PINNED** — do NOT upgrade. ROS2 compatibility depends on this exact version.
- `.venv` is created with `--system-site-packages` to access ROS2 Python packages.

## 8. ROS2 Conventions

- Default distro: detected via `$ROS_DISTRO` environment variable — do not hardcode a distro name.
- Workspace layout: `<workspace>/src/<package>/<package>/` for node Python files.
- **Node naming convention**: Python files generated by nbdev become ROS2 nodes. The `default_exp` in the notebook MUST match the desired `ros2 run` command:
  - `ros2 run <package> camera_node` → notebook must have `#| default_exp camera_node`
  - The exported file will be `<package>/camera_node.py`
  - Always name the notebook and its `default_exp` to match how you want to call it with `ros2 run`
- Every node file **MUST** define `def main(args=None):` — `universal_setup.py` auto-discovers entry points by scanning for this pattern.
- Build command: `bash scripts/compile_pkg.sh` (generates `setup.py` + colcon build)
- Run command: source workspace overlay, then `ros2 run <package> <node>`

⚠️ **Never run scripts or create ROS2 workspaces/packages unless explicitly asked by the user.** Only execute `compile_pkg.sh`, `generate_ros2_ws.zsh`, `mk_ros2pkg_opencv.zsh`, `mk_ros2_pkg.sh`, or `run_cam.zsh` when the user explicitly requests it.

Before any commit that touches notebooks or exported code:

```bash
uv run nbdev-prepare    # = export + test + clean + readme
```

⚠️ **This is mandatory.** There is no CI yet — all validation is manual.
